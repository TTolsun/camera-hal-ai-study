선택 방식과 운영 기준

이번 페이지는 누가 정한 순서를 따라가는 문서가 아니라, 각자가 관심 있는 아이템을 고르고 실제 데모 가능한 범위를 잡기 위한 후보 목록입니다. 목록은 고정된 것이 아니며, 필요하면 새로운 아이템을 계속 추가할 수 있습니다.

원칙 1

관심 있는 아이템을 자유롭게 선택

모든 후보를 한 줄로 세우기보다, 본인이 실제로 해보고 싶은 아이템을 선택하고 데모 가능한 범위를 직접 잡는 방식으로 운영합니다.

원칙 2

MM은 본인 상황에 맞춰 조정

문서의 MM은 참고값입니다. 실제 진행 범위, 자동화 깊이, 데모 수준에 맞춰 각자 현실적인 MM으로 다시 산정합니다.

원칙 3

아이템 중복은 가급적 줄이기

동일 아이템에 관심이 몰리더라도 최종 진행 범위가 지나치게 겹치지 않도록 조율합니다. 필요하면 유사 아이템이나 확장 아이템으로 자연스럽게 분산합니다.

원칙 4

원하는 아이템은 언제든지 추가 가능

현재 후보에 없는 아이디어도 스터디 목적에 맞고 데모 가능한 범위로 정리할 수 있다면 언제든지 새로운 아이템으로 추가할 수 있습니다.

핵심은 “정해진 후보 안에서만 고르는 것”이 아니라, 본인이 끝까지 만들 수 있는 범위를 잡고 실제 결과물을 남기는 것.

최종 산출물 기준

각 아이템은 단순 조사로 끝내지 않고, 실행 가능한 데모 또는 문서화된 결과물을 남기는 것을 기준으로 합니다.

필수

README 또는 사용 가이드

입력, 실행 방법, 출력 예시, 한계를 짧게 정리합니다.

필수

데모 결과

스크립트, HTML, Markdown, JIRA 초안 등 눈으로 확인 가능한 결과물을 남깁니다.

권장

전체 아이템 리스트

아래 후보들은 스터디에서 검토하거나 선택해 진행할 수 있는 아이템입니다. 표시된 MM은 참고값이며, 실제 담당자는 본인의 상황에 맞춰 범위를 줄이거나 넓혀 다시 산정하면 됩니다. 목록에 없는 아이디어도 원하면 언제든지 새로운 아이템으로 추가할 수 있습니다.

AI Item 추가 요청

아이템은 많을수록 좋은 것이 아니라, 실제로 끝까지 데모 가능한 범위로 정리되는 것이 중요.
AI Item 제안하기
Item 01

리뷰 파이프라인 자동화

리뷰/협업 2.5~3.0 MM

요약 사전점검과 리뷰어 할당 통합

핵심 MVP Gerrit 변경을 입력받아 코딩룰 점검, 커밋 설명문 초안, 분할/병합 권고를 만들고, HAL CI 완료 후 오너십 데이터를 기반으로 리뷰어를 최대 3명 추천하거나 자동 할당한다.

입출력 입력: Gerrit 변경/commit 목록, 코딩룰 문서, HAL CI 결과, 코드 오너십 데이터 / 출력: 사전점검 결과, 커밋 설명문 초안, 리뷰어 추천 또는 할당 결과

Item 02

이슈 초도분석 도우미

디버깅/분석 2.0~2.5 MM

요약 로그 분석, 유사 사례, assign 통합

핵심 MVP JIRA 이슈와 첨부 로그를 읽어 실패 시점과 원인 후보를 정리하고, 과거 유사 이슈 Top 5와 담당자 추천 또는 assign 결과를 함께 제시한다.

입출력 입력: JIRA 이슈, 첨부 로그, 과거 JIRA 이력, 코드 오너십 데이터 / 출력: 실패 시점, 원인 후보, 유사 사례 5건, 담당자 추천/할당 결과

Item 03

변경 영향도 시뮬레이터

디버깅/분석 2.5~3.0 MM

요약 패치 영향 범위와 위험 포인트 예측

핵심 MVP Gerrit 변경이나 코드 diff를 입력받아 영향 받는 모듈, 의심 경로, 먼저 돌려야 할 테스트, 함께 봐야 할 리뷰 포인트를 정리한다.

입출력 입력: Gerrit 변경/코드 diff, 관련 코드, 테스트 정보, JIRA/Confluence 문서 / 출력: 영향 모듈 목록, 위험 포인트, 추천 테스트, 리뷰 포인트 요약

Item 04

자율 디버깅 루프

디버깅/분석 3.0~4.0 MM

요약 이슈부터 수정안·검증까지 한 바퀴

핵심 MVP JIRA 이슈와 로그를 입력받아 원인 후보를 정리하고 수정안 초안을 만든 뒤, HAL CI 또는 테스트 결과를 붙여 검증 요약까지 한 번에 생성한다. 자동 submit은 제외한다.

입출력 입력: JIRA 이슈, 첨부 로그, 관련 코드, HAL CI 또는 테스트 결과 / 출력: 원인 후보, 수정안 초안, 검증 요약, 다음 액션 제안

Item 05

Action Item → JIRA 도우미

업무 효율화 1.5~2.0 MM

요약 회의/메일 내용을 JIRA로 전환

핵심 MVP 회의록, 메일, 메신저 내용을 읽고 Action Item을 뽑아 신규 JIRA 초안 또는 기존 JIRA 업데이트 초안을 만든다.

입출력 입력: 회의록/메일/메신저 내용, 기존 JIRA 후보, 관련 Confluence / 출력: Action Item 목록, 신규 JIRA 초안, 기존 JIRA 업데이트 초안

Item 06

코드 오너십 관리 서비스

공통기반 2.0~2.5 MM

요약 기능/모듈 담당자 관리 기반

핵심 MVP 폴더/모듈 단위로 최근 기여도를 계산해 주 담당자, 보조 담당자, 리뷰 후보를 만들고, '이 기능 담당 누구야?' 같은 질문에 답할 수 있게 주기적으로 갱신한다.

입출력 입력: git log/blame, JIRA/Gerrit 이력, 폴더/모듈 매핑, 수동 예외 규칙 / 출력: 담당자 목록, 리뷰 후보, 담당자 질의응답 결과

Item 07

Camera HAL 지식 질의 시스템

공통기반 2.5~3.0 MM

요약 코드·문서·이슈를 연결한 자연어 질의

핵심 MVP 코드, JIRA, Gerrit, Confluence 정보를 연결해 '이 기능 담당 누구야?', '이 함수와 연관된 이슈 뭐야?' 같은 질문에 답하는 질의 시스템을 만든다.

입출력 입력: 코드 저장소, JIRA, Gerrit, Confluence, 코드 오너십 데이터 / 출력: 자연어 질의응답 결과, 관련 코드/문서/이슈 링크

Item 08

릴리즈 리스크 요약기

형상/릴리즈 1.5~2.0 MM

요약 릴리즈 리스크 2주 단위 요약

핵심 MVP 최근 2주 develop 변경을 모아 관련 JIRA/Confluence와 연결하고, 변경 요약·고위험 항목·확인 포인트를 정리해 지정 JIRA에 올린다.

입출력 입력: develop branch commit 내역, 관련 JIRA/Confluence, 결과를 올릴 JIRA / 출력: 릴리즈 리스크 요약, 고위험 항목, JIRA 업데이트 결과

Item 09

Camera HAL SW 뉴스레터 자동 생성기

지식공유 1.5~2.0 MM

요약 2주 단위 기술동향 뉴스레터 생성

핵심 MVP 2주에 한 번 Android 카메라 관련 변경사항, C++ 문법/표준 트렌드, AI 관련 주요 흐름을 수집해 Camera HAL 개발자 관점으로 요약한 뉴스레터를 자동 생성한다. 필요하면 Confluence나 공유용 HTML/Markdown으로 내보낸다.

입출력 입력: Android 공식 문서/릴리즈 노트, C++ 관련 공식/주요 기술 자료, AI 관련 주요 기술 자료, 기존 뉴스레터 템플릿 / 출력: 2주 단위 뉴스레터 초안, 핵심 변경사항 요약, 링크 모음, 추천 스터디 주제

Item 10

테스트 설계 보조 도우미

품질/테스트 2.0~2.5 MM

요약 누락 테스트 추천과 TC 초안 통합

핵심 MVP 코드 변경이나 JIRA 이슈를 입력받아 가장 효과가 큰 테스트 누락 포인트를 추천하고, 재현 절차와 positive/negative TC 초안을 함께 만든다.

입출력 입력: 코드 변경 내역 또는 JIRA 이슈, 기존 테스트 코드 / 출력: 추천 테스트 항목, 재현 절차, positive/negative TC 초안

Item 11

구현 보조 도우미

개발 생산성 2.0~2.5 MM

요약 레거시 이해와 코드 초안 생성 통합

핵심 MVP 함수/코드 구간을 설명하고 영향 범위를 정리한 뒤, 요구사항을 받아 기존 구조와 룰에 맞는 코드 초안을 만든다.

입출력 입력: 함수/코드 구간 또는 파일, 요구사항, 기존 코드, 아키텍처 문서, 룰, 관련 JIRA/Confluence / 출력: 코드 설명, 영향 범위, 추가 질문, 코드 초안, TODO 목록

Item 12

보고/고객 대응 초안 생성기

문서화 1.0~1.5 MM

요약 보고/고객 대응 문서 초안 생성

핵심 MVP 이슈 현황과 관련 코드/문서를 바탕으로 내부 보고 초안이나 고객 대응 초안을 만든다.

입출력 입력: 이슈 현황, 관련 코드, JIRA, Confluence / 출력: 보고 초안, 고객 대응 초안, 핵심 근거 요약

Item 13

사양서 변경점 비교 분석기

문서화 1.5~2.0 MM

요약 HW 변경 기반 SW note 초안 생성

핵심 MVP HW 변경 문서를 비교해 SW change note 초안을 만들고, 필요하면 Camera Driver code까지 참고한다.

입출력 입력: HW Spec, HW change note, JIRA, Confluence, 필요 시 Driver code / 출력: SW change note 초안, 영향 포인트, 확인 필요 항목

Item 14

WSL Setup 도우미

온보딩 1.0~1.5 MM

요약 개발환경 셋업과 초기 트러블 대응

핵심 MVP WSL 기반 Camera HAL 개발환경 절차를 안내하고, 셋업 중 발생한 오류의 원인을 사내 문서 우선으로 찾아 해결 가이드를 제시한다.

입출력 입력: 셋업 요청, 오류 로그/증상, JIRA, Confluence, 필요 시 웹 검색 결과 / 출력: 셋업 가이드, 체크리스트, 오류 원인 후보, 해결 가이드

Item 15

이슈 재발 방지 대책 생성기

업무 효율화 0.5~1.0 MM

요약 이슈 재발 방지 대책 정리

핵심 MVP JIRA 이슈의 원인 분석과 수정 내역을 바탕으로 재발 방지 액션과 리뷰/테스트 체크리스트를 만든다.

입출력 입력: JIRA 이슈, 원인 분석, 수정 내역 / 출력: 재발 방지 액션, 리뷰 체크리스트, 추가 테스트 포인트

Item 16

SAM 개선 포인트 도출기

품질/정적분석 1.5~2.0 MM
