운영 원칙

Practical AI Study는 AI Agent를 업무에 바로 쓰기 전에, 공통 사용 기준과 검증 습관을 먼저 맞추는 과정입니다.

원칙 1

작게 시작하기

처음부터 큰 기능 구현을 맡기기보다, 코드 흐름 요약, 로그 정리, 테스트 아이디어 도출처럼 실패해도 되돌리기 쉬운 작업부터 시작합니다.

원칙 2

컨텍스트를 먼저 설계하기

AI Agent에게 어떤 파일, 로그, 설계 문서, 규칙 문서를 읽힐지 먼저 정합니다. 좋은 답변은 좋은 질문보다 좋은 입력 구조에서 나옵니다. 아쉽지만 기계도 눈치로 일하지 않습니다.

원칙 3

반드시 검증하기

AI가 만든 결과는 초안으로 보고, diff 확인, 빌드, 테스트, 로그 검증을 통해 사람이 최종 판단합니다. 빠른 자동화보다 안전한 반복 가능성이 우선입니다.

핵심은 AI에게 일을 던지는 것이 아니라, AI가 낸 결과를 검증 가능한 단위로 쪼개고 안전하게 반복하는 사용법을 익히는 것.

스터디에서 맞출 공통 프레임

스터디에서는 도구별 사용법보다 먼저, 어떤 방식으로 AI Agent에게 일을 맡기고 확인할지에 대한 공통 기준을 맞춥니다.

컨텍스트 설계

AI Agent에게 무엇을 읽히고, 어떤 제약을 주고, 어떤 산출물을 기대할지 먼저 정의합니다. Camera HAL 업무에서는 코드, 로그, 설계 문서, 규칙 문서를 어떻게 조합하느냐가 결과 품질을 크게 좌우합니다.

Rules Logs Design Docs Expected Output

실전 가드레일

권한, 자동 개입 규칙, 민감 경로, 검증 명령, 산출물 정리 방식을 함께 잡습니다. 어디까지 맡기고 어디서 멈춰야 하는지 정하지 않으면, AI Agent는 성실하게 이상한 방향으로 달릴 수 있습니다.

Permissions Hooks Verify Handoff

작업 단위 쪼개기

큰 요청을 한 번에 맡기지 않고, 이해 → 계획 → 수정 → 검증 → 정리 단계로 나눕니다. 이렇게 해야 결과를 사람이 추적할 수 있고, 잘못된 변경도 빠르게 되돌릴 수 있습니다.

Understand Plan Implement Review

산출물 남기기

좋은 사용 사례, 실패 사례, 프롬프트, 체크리스트, handoff 문서를 남겨 다음 작업에 재사용합니다. 한 번 배운 내용을 매번 새로 설명하는 일은 인류 문명의 낭비입니다.

Checklist Prompt Decision Log Template

학습 모듈

Practical AI Study의 학습 모듈은 AI Agent를 안전하게 업무에 적용하기 위한 기본기 중심으로 구성합니다.

Module 1. AI Agent 기본 활용

입문

AI Agent를 단순 채팅 도구가 아니라, 문서를 읽고 정리하고 코드 작업을 보조하는 실무형 도구로 이해합니다.

  • AI Agent가 잘하는 일과 조심해야 할 일
  • 질문형 사용과 구현형 사용의 차이
  • 작고 되돌리기 쉬운 첫 과제 고르기

Module 2. 첫 실습과 기본 운영 방식

실습

설치보다 중요한 것은 어떤 파일을 먼저 읽히고, 어떤 기준으로 결과를 검증할지 정하는 것입니다. 초반에는 안전한 첫 사용 경험을 만드는 데 집중합니다.

  • 기본 규칙 파일과 작업 범위 정리
  • 작은 예제로 plan → implement → verify 흐름 체험
  • 예시: capture request / result flow 요약, 기존 디버깅 로그 정리

Module 3. 파일 구조와 템플릿

구조화

배경 정보, 작업 파일, 결과 파일, 템플릿을 분리해 관리하고, 나중에 다시 사용할 수 있는 구조를 만듭니다.

  • background, task, output, reference 파일 구분
  • template과 reference의 차이 이해
  • plan, handoff, decision log 역할 정리

Module 4. 권한과 검증

가드레일

빠르게 돌리는 것보다 안전하게 반복 가능하게 만드는 것이 중요합니다. 권한 범위, 민감 경로, 검증 기준, 자동화 규칙을 함께 다룹니다.

  • 개인 설정과 팀 공통 설정 분리
  • 고위험 작업 경계와 승인 기준 설계
  • hooks와 verify를 통한 반복 검증 자동화

Module 5. IDE 연동과 실전 워크플로우

개발 흐름

VS Code 연동, diff 확인, 계획 검토, 여러 파일 수정, handoff 정리처럼 실제 개발 흐름에서 자주 쓰는 기능을 실습합니다.

  • IDE 연동과 CLI 활용 차이
  • 여러 파일 수정 시 계획 검토와 diff 확인
  • 예시: Gerrit 리뷰 전 변경 요약, 레거시 C++ 코드 리팩토링 초안 작성

Module 6. Skills, MCP, 운영 자산화

확장

반복 절차는 Skill로 정리하고, 외부 시스템 연동은 MCP 같은 연결 구조로 이해합니다. 자주 쓰는 사용법을 개인 요령이 아니라 팀 자산으로 남기는 것이 목표입니다.

  • 반복 절차를 재사용 가능한 형태로 정리
  • 외부 시스템 연결 개념 이해
  • 체크리스트, 규칙 문서, handoff 템플릿 자산화

회차 운영 방식

매 회차는 강의만 듣고 끝나는 방식이 아니라, 짧은 교육과 실습, 결과 공유가 이어지는 구조로 운영합니다.

기본 진행 흐름

  • 지난 회차 실습 결과와 막힌 지점 공유
  • 이번 회차 핵심 개념 또는 기능 설명
  • 작은 예제로 AI Agent 사용 흐름 실습
  • 다음 회차까지 적용할 작은 과제 정리

실습 기준

  • 한 번에 큰 변경을 맡기지 않고 작은 단위로 요청
  • AI가 만든 결과는 diff, 빌드, 테스트, 로그로 확인
  • 좋았던 프롬프트와 실패 사례를 함께 기록
  • 반복 가능한 절차는 체크리스트나 템플릿으로 정리
